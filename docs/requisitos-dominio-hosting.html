Requisitos para publicar el sistema: dominio y hosting

Para que la facturación electrónica, el pago con tarjeta y el seguimiento del repartidor en vivo funcionen de verdad, el sistema tiene que estar publicado en internet de forma segura. Para eso se necesitan dos cosas: un dominio (el nombre web) y un hosting (el servidor donde vive el sistema).

Parte 1 — Dominio

El nombre web del sistema Pago 1 vez al año

Es la dirección por la que se entra al sistema y por la que tus clientes abren el link de seguimiento (ej. lavixa.pe).

1

Elegir el nombre

Decide cómo quieres que se llame la web de tu negocio. Puede ser el nombre de la lavandería. Ejemplos: lavixa.pe, lavanderia-lavixa.com, mi-lavanderia.pe.

Tú solo eliges el nombre; del registro técnico se encarga el equipo.

2

La terminación (.com o .pe)

Las dos sirven. .com es la más conocida y económica; .pe transmite que es un negocio peruano.

Costo referencial: .com ~S/ 40–60 al año · .pe ~S/ 80–130 al año.

3

A nombre de quién queda

Se recomienda registrarlo a tu nombre / de tu negocio, para que el dominio sea siempre tuyo y no dependas de nadie.

Dónde se compra: registradores como Namecheap, GoDaddy, Hostinger, o proveedores locales. Cualquiera sirve; funciona con cualquier hosting.

Lo que necesito de ti

  • El nombre que quieres para la web.
  • Tu preferencia de terminación (.com o .pe) — o lo recomiendo yo.
Parte 2 — Hosting

El servidor donde vive el sistema Pago mensual

Es como el “local alquilado” en internet: una computadora prendida 24/7 que corre el sistema para que esté siempre disponible.

1

Aprobar el costo mensual

El hosting se paga todos los meses. Para una lavandería con una o pocas sedes, un servidor pequeño alcanza de sobra.

Costo referencial: ~S/ 40–150 al mes. Se empieza con el plan más pequeño y se agranda solo si hace falta.

2

El candado de seguridad (HTTPS) GRATIS

Es el candado que aparece al lado de la dirección web y que da seguridad. No se compra: se instala gratis. Sin él, ni la facturación ni el pago con tarjeta ni el mapa del repartidor funcionan.

Se encarga el equipo al momento de publicar; no es un trámite tuyo.

3

A nombre de quién queda

Igual que el dominio: lo ideal es que la cuenta del hosting esté a tu nombre, para que el servicio y los datos del negocio sean tuyos.

Se decide contigo si lo pagas directo o me lo reembolsas.

Lo que necesito de ti

  • Aprobar el costo mensual del hosting (con el plan pequeño arrancamos).
  • Decidir si la cuenta va a tu nombre y quién la paga.
Parte 3 — Recomendaciones

Qué comprar para que sea compatible

No cualquier hosting sirve: el sistema usa .NET, base de datos SQL Server y una web en Angular. Esta parte es para que elijas (o le muestres a quien te venda el hosting) lo correcto y no gastes en algo que no funcione.

1

Tipo de servicio: un “VPS” o servidor en la nube

Necesitas un servidor propio (VPS o cloud), no un hosting compartido. En un VPS podemos instalar .NET y SQL Server, que es lo que este sistema requiere.

Proveedores que sirven: DigitalOcean, Contabo, Hostinger (plan VPS), Microsoft Azure, AWS Lightsail, o un proveedor local que ofrezca “VPS Windows”.

2

Características mínimas a pedir

Muéstrale esta tabla al proveedor. Con esto corre el sistema, la base de datos y la web sin problemas para empezar:

Sistema operativoWindows Server (2019 o 2022). Es lo más directo para SQL Server.
Memoria (RAM)4 GB como mínimo (8 GB recomendado si crece).
Procesador2 núcleos (vCPU) o más.
Disco80 GB SSD o más (rápido, tipo SSD/NVMe).
Base de datosCompatible con SQL Server (sirve la versión gratuita SQL Server Express).
ExtrasIP fija incluida y respaldos/backups automáticos.
3

Sobre la base de datos (SQL Server)

El sistema guarda todo (pedidos, clientes, caja) en SQL Server. Para empezar alcanza SQL Server Express, que es gratuito; solo si el negocio crece mucho se evalúa una versión mayor. No necesitas comprar licencia de base de datos al inicio.

Se instala en el mismo servidor, no hay que contratarlo aparte.

4

Alternativa “sin mantenimiento” (opcional)

Si prefieres olvidarte de administrar el servidor, existe la opción de una nube administrada (por ejemplo Microsoft Azure con “App Service” + “Azure SQL”). Es más cómoda pero suele costar más al mes.

Recomendación: para arrancar, el VPS Windows es lo más económico y suficiente.

Qué NO comprar

Evita el “hosting compartido” tipo cPanel (los planes baratos pensados para páginas WordPress/PHP). No pueden correr .NET ni SQL Server, así que el sistema no funcionaría ahí. Si el proveedor solo ofrece eso, no es el adecuado.

Resumen para pedirle al proveedor

  • “Quiero un VPS con Windows Server, 4 GB de RAM, 2 vCPU, 80 GB SSD, con IP fija.”
  • “Debe permitir instalar .NET y SQL Server Express.”
  • “No me sirve hosting compartido / cPanel.”
!

Los montos son referenciales (mercado peruano, aproximados). Conviene cotizar 2 o 3 proveedores antes de decidir. Toda la configuración técnica (instalar el sistema, la base de datos y el candado de seguridad) la hace el equipo; de tu lado solo es elegir el nombre y aprobar el hosting.

Preguntas frecuentes

¿Por qué necesito esto si el sistema ya funciona?

Hoy funciona en la red local del negocio. Para que los clientes abran el link de seguimiento desde su casa, y para que la facturación y el pago con tarjeta operen, el sistema debe estar publicado en internet con el candado de seguridad. Eso requiere dominio + hosting.

¿El dominio y el hosting son lo mismo?

No. El dominio es el nombre (se paga una vez al año). El hosting es el servidor donde vive el sistema (se paga cada mes). Se usan juntos.

¿Cuánto gasto en total al inicio?

Aproximado: el dominio una vez al año (S/ 40–130) y el hosting mensual (desde ~S/ 40). El candado de seguridad es gratis. Son montos referenciales.

¿Puedo elegir cualquier proveedor barato?

No cualquiera: tiene que permitir .NET y SQL Server (un VPS, no hosting compartido). En la Parte 3 está exactamente qué pedir para no equivocarte.
